# ADR 0004: Featured project framing

## Status

Accepted

## Context

Projects should be framed as evidence of product and engineering judgment, not just screenshots or technology demos.

## Decision

Use the following framing for featured projects.

### SwiftDispatch

Frame as dispatch software for small HVAC teams outgrowing calls, texts, and whiteboards.

Emphasize:

- Small-business operations software
- Live dispatch workflow
- Technician SMS updates
- Customer intake
- Quote approval
- Analytics
- Supabase/Twilio/Vercel integration
- Background SMS processing
- Job status/state-machine thinking

### Serpent Ascension League

Frame as a community league operations platform, not simply a gaming website.

Emphasize:

- Public league hub
- Standings, schedule, teams, players, watch pages
- Admin tooling
- Registrations
- Discord authentication
- Draft tooling
- Testing and production-readiness audit work
- Domain modeling for seasons, divisions, matches, rosters, and draft flows

### ThreeTails Booking

Frame as a custom booking and payment workflow for a cat cafe concept.

Emphasize:

- Client-style workflow design
- Rules agreement flow
- Smart booking/calendar logic
- Party-size/time-of-day constraints
- Square checkout
- Email confirmations
- Custom business-specific UX

## Consequences

- Project cards and case studies should lead with problem/user/value before tech stack.
- SwiftDispatch and ThreeTails should have dedicated case study pages.
- Serpent Ascension League should also be treated as a dedicated case study because of its application depth.
