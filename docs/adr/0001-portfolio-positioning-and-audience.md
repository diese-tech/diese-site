# ADR 0001: Portfolio positioning and audience

## Status

Accepted

## Context

The existing site is a lightweight Discord automation/service landing page. The next version should become a proper portfolio that can be shared with future employers and listed on LinkedIn. It should still preserve the "Diese" identity, but the primary goal is employer-facing credibility rather than selling Discord setup services.

The target audience is primarily recruiters and hiring managers, with secondary relevance for startup/freelance opportunities. The portfolio should support upward mobility into frontend, product engineering, AI applications, or frontend/full-stack hybrid roles.

## Decision

Position the portfolio around this core headline:

> I build practical web applications for real-world workflows.

Use this professional role framing:

> Product-minded frontend developer

Use this supporting message:

> I design and ship polished interfaces, useful automation, and full-stack tools for small teams, communities, and operational problems — from dispatch software to league platforms and custom booking systems.

The portfolio should emphasize practical product-building, real users, workflow design, and frontend polish rather than a generic list of technologies or a purely visual 3D showcase.

## Consequences

- The homepage should lead with projects and outcomes, not services or pricing.
- Copy should describe real-world problems, users, constraints, integrations, and results.
- AI and 3D should support the product story rather than dominate it.
- The portfolio should be credible for full-time roles while remaining open to freelance work.
